How I configured neovim to be simple yet powerful

The full code for my neovim configuration can be found here.

Introduction

Setting up nvim can be truly overwhelming due to the sheer number of plugins and the seemingly limitless customization options. I wanted a setup that's relatively straightforward but still stacks up against an IDE (Integrated Development Environment) like JetBrains Rider.

File Structure


.config/nvim
    ⎥lua
    ⎥   ⎥lazy
    ⎥   ⎥   ⎥->  lazy.lua
    ⎥   ⎥core
    ⎥   ⎥   ⎥->  init.lua
    ⎥   ⎥   ⎥->  options.lua

    ⎥      ⎥->  init.lua

    ⎥      ⎥->  nvim-surround.lua
    ⎥      ⎥->  nvim-tree.lua
    ⎥      ⎥->  telescope.lua
    ⎥->  init.lua

Explaination

  • init.lua - This is the file neovim calls upon being launched, this contains a bunch of require() statements calling each induvidual componenet

  • /lua/lazy/lazy.lua - Bootstrapping the Lazy.nvim plugin manager

  • /lua/core/ - This folder contains non plugin related configs like:
    • init.lua - File called by primary init.lua
    • options.lua - Contains some regular vim options
    • keymaps.lua - Contains some regular vim operations to certain keybinds

  • /lua/plugins/ - This folder contains the configs for induvidual plugins

Primary init.lua file

Over here I call the /lua/core/init.lua file which then calls the /lua/core/options.lua & the /lua/core/keymaps.lua file. We also initiate the Lazy.nvim plugin manager.
